## How to use Vectra

Vectra is built for speed and visual precision. Your typical workflow in Vectra will look like this:

1. **Bootstrap with the Marketplace:** Open the **Marketplace** `🛍️` to browse high-fidelity sections (Heros, Footers, Pricing grids). Find what you like and insert it.
2. **Build with AI:** Use the **AI Generation** `✦` tool to describe new sections using natural language.
3. **Visual Customization:** Click any element to edit its properties, typography, and styling in the **Design Panel**.
4. **Iterative Refinement:** Drag and drop elements on the **Canvas** to rearrange them, zoom in and out with `Ctrl + Scroll`.
5. **Deployment:** Click **Export** to get a clean React + Tailwind project ready to deploy on Vercel, Netlify, or your own server.

## Hybrid Engine Architecture (Advanced)

Behind the scenes, Vectra uses a powerful hybrid engine that separates concerns between visual state and performance:

<Callout type="info">
  **TypeScript layer** — handles UI state, event dispatch, and the React canvas. This is the "face" of the Studio.<br/><br/>
  **Rust WASM layer** — performs all the heavy lifting: SWC code compilation, undo/redo history, and layout snapping. It keeps the Studio fast even with hundreds of elements.
</Callout>

### Core Concepts

- **VectraNode** — Every element is a serializable object.
- **Intent Events** — Every drag or resize is an Intent Event that changes the project state.
